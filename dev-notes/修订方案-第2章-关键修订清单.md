- 验收员批评：**新手可能把整个 withRouting() 删掉重写，导致原有路由丢失**

### 修订方案

**替换第 144-158 行**为完整的"修改前后对照"：

```markdown
### 2.3.2 修改 bootstrap/app.php（挂载 admin / api 路由）

**找到 `withRouting()` 调用**（约在第 14-18 行），把它改成下面这样：

**修改前**（Laravel 12 默认）：
```php
->withRouting(
    web: __DIR__.'/../routes/web.php',
    commands: __DIR__.'/../routes/console.php',
    health: '/up',
)
```

**修改后**（加上 admin / api 路由，用 `then:` 参数）：
```php
->withRouting(
    web: __DIR__.'/../routes/web.php',
    commands: __DIR__.'/../routes/console.php',
    health: '/up',
    then: function (): void {
        Route::prefix('api/admin')->name('admin.')->group(base_path('routes/admin.php'));
        Route::prefix('app')->name('app.')->group(base_path('routes/api.php'));
    },
)
```

> - 保留原有的 `web`、`commands`、`health` 参数

## 🟠 P1-2：iResource 宏必须给完整上下文

### 当前问题
- 第 160-173 行的 iResource 宏代码缺文件上下文
- 验收员批评：**缺少 use 语句，新手会报 Class 'Route' not found**

### 修订方案

**替换第 160-173 行**为：

```markdown
### 2.3.3 注册 iResource 宏（在 AppServiceProvider::register() 里）

**第一步：在 `app/Providers/AppServiceProvider.php` 文件顶部补齐 use 语句**

在 `namespace App\Providers;` 下方，补充：

```php
use Illuminate\Support\Facades\Route;
```

**第二步：在 `register()` 方法里注册宏**

找到 `AppServiceProvider` 类的 `register()` 方法（约在第 17-23 行），在方法内添加：

```php
public function register(): void
{
    // 原有内容（如 $this->app->singleton(...)）保留

    // 注册 iResource 宏（moo-scaffold 生成的路由要用）
    Route::macro('iResource', function (string $name, string $controller): void {
        /** @var \Illuminate\Routing\Router $this */
        $this->get("{$name}", "{$controller}@index")->name("{$name}.index");
        $this->get("{$name}/create", "{$controller}@create")->name("{$name}.create");
        $this->post("{$name}", "{$controller}@store")->name("{$name}.store");
        $this->get("{$name}/trashed", "{$controller}@trashed")->name("{$name}.trashed");
        $this->get("{$name}/{id}", "{$controller}@show")->name("{$name}.show");
        $this->get("{$name}/{id}/edit", "{$controller}@edit")->name("{$name}.edit");
        $this->patch("{$name}/{id}", "{$controller}@update")->name("{$name}.update");
        $this->put("{$name}/{id}", "{$controller}@update")->name("{$name}.update.put");
        $this->delete("{$name}/{id}", "{$controller}@destroy")->name("{$name}.destroy");
        $this->delete("{$name}/forever/{id}", "{$controller}@forceDestroy")->name("{$name}.force_destroy");
    });
}
```

> **为什么写在 `register()` 而不是 `boot()`？** 第 7 章的 moo-system 包会在其 provider 的
> `boot()` 方法里加载路由文件，那时会用到这个宏——必须在包 boot 之前注册（`register` 先于 `boot`）。
```

## 🟡 P2-7：坑 4（多 worker）移到第 2 章开头

### 当前问题
- 第 301-306 行的"坑 4"在第一次用调试器时才提，但启动方式应该更早说明

### 修订方案

**删除第 301-306 行的"坑 4"**，改为在 **2.1 节末尾**（第 87 行 `composer update` 之后）补充：

```markdown
### 2.1.5 ⚠️ 启动方式变更（多 worker + --no-reload）

从本章起，启动命令必须加 `PHP_CLI_SERVER_WORKERS=4` 和 `--no-reload`：

```bash
# 如果第 1 章的服务器还在跑，先 Ctrl+C 停掉
PHP_CLI_SERVER_WORKERS=4 php artisan serve --host=127.0.0.1 --port=8088 --no-reload
```

**原因**：scaffold 的接口调试器（`/scaffold/api/request`）内置代理功能，会回调同一个
服务器（`/scaffold/api/proxy`）——单线程 `serve` 只有一个 worker，自己调自己会死锁。
`--no-reload` 禁止文件变化时自动重启（配合多 worker 使用）。

**症状**：用单 worker 启动时，访问调试器发送请求会无限转圈，浏览器开发者工具看到请求卡在 pending。
```

### 影响范围
🔸 正文 + 引导器（第 2 章新增步骤）
