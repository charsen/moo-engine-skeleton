## 🟠 P1-2：1.6 启动方式简化（多 worker 移到第 2 章）

### 当前问题
- 验收员批评：第 1 章过度前置了第 2 章才需要的多 worker 启动方式

### 修订方案

**1.6 保持最简启动**（第 144-158 行替换为）：

```markdown
## 1.6 启动并真机访问

启动开发服务器（端口与 1.4 的 `APP_URL` 保持一致，教程统一用 8088）：

```bash
php artisan serve --host=127.0.0.1 --port=8088
```

> **注意**：`php artisan serve` 是前台进程，会一直占着当前终端（`Ctrl+C` 停止）。
> 浏览器访问不受影响，但下面的命令行自检需要**另开一个终端窗口**再执行。

> 端口被占时如何排查：
> ```bash
> lsof -nP -iTCP:8088 -sTCP:LISTEN     # 看谁在用 8088
> ```
```

**在第 2 章 2.1 节末尾**（接入 scaffold 后）插入新小节：

```markdown
### 2.1.5 ⚠️ 启动方式变更（多 worker + --no-reload）

从本章起，启动命令必须加 `PHP_CLI_SERVER_WORKERS=4` 和 `--no-reload`：

```bash
# 如果第 1 章的服务器还在跑，先 Ctrl+C 停掉
PHP_CLI_SERVER_WORKERS=4 php artisan serve --host=127.0.0.1 --port=8088 --no-reload
```

**原因**：scaffold 的接口调试器（`/scaffold/api/request`）内置代理功能，会回调同一个
服务器（`/scaffold/api/proxy`）——单线程 `serve` 只有一个 worker，自己调自己会死锁。
`--no-reload` 禁止文件变化时自动重启（配合多 worker 使用）。

**症状**：用单 worker 启动时，访问调试器发送请求会无限转圈，浏览器开发者工具看到请求卡在 pending。
```

### 影响范围
🔸 正文 + 引导器（第 1 章步骤简化、第 2 章新增步骤）
