# notes.md — 项目踩坑随手记

本文件是本项目的「踩坑底账」：把踩过的坑一条条记下来，给未来的自己、也给接手的人省时间。

**记什么**——每条坑记清四要素：
- **症状**：当时看到的现象 / 报错信息
- **根因**：定位到的真正原因
- **解法**：怎么修好或绕过的
- **日期**：踩坑当天

**什么时候记**——踩坑当下随手记，别等事后补，细节当天最清楚；做升级、迁移、或接手项目前先通读一遍，能少踩很多重复的坑。
