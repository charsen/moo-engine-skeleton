The pain

每个团队都在重复交的"三笔学费"

这套工具不是为了炫技,是先有这些痛,才一个个补上的。

建

口径漂移

一张表 8~15 个字段,要在 Model / Migration / Request / 文档 / 权限 六七处重复声明 —— 改一处忘改另一处,三处不一致是常态。

看

线上黑盒

异常散落各机器日志、靠人工巡查;生产问题常常等客户投诉了才发现,而且缺堆栈、缺现场,复现成本极高。

协作

报障靠追问

测试在群里发张截图配半页描述,开发追问"什么请求、报什么错";消息被淹没、没指派、没状态,一条 bug 沟通成本翻倍。

Why it works

六个让它不一样的点

不是堆功能,是几条贯穿始终的取舍,让这套工具"省事且不漂移"。

单一事实源

一份 YAML 同时派生校验、文档、调试参数、表单控件 —— 改一处,四端同步,天然不漂移。这是通用工具结构上拿不到的。

YAML → 校验 → 文档 → 调试 → 前端

全栈监控闭环

前端 + 后端 + 人工三路汇进一个云,共用项目 / token / 告警渠道。

报障即协作

截图打包现场进待办,带指派 / 闭环 / 进度回查 —— 比微信群截图有效得多。

AI · MCP 桥

线上错误经 MCP 拉回代码仓库修复、再回写已解决,让"看"和"建"闭环。

数据自主 · 自托管

监控数据落在自己服务器,不出企业、不按量计费,合规与成本上的实在好处。

分层复用

scaffold 当引擎、system 当基座、radar 长出行业应用 —— 一份基建供多个项目。

Built on purpose

为什么"明明有现成的,还自己造"

判断只有一条:当"和自己代码共享同一份 schema"能带来通用工具拿不到的好处时,自造才值。 两个例子。

scaffold · 数据库设计器

改完表,迁移自己写好

  • ▸改完自动出 migration,不用手写 up()/down()
  • ▸字段改名走 renameColumn 保数据,不是 drop+add 丢数据
  • ▸删字段前全仓 grep 查引用,分"自动清 / 手动清"
  • ▸schema 进 git + 基线快照 —— 团队改表在 git 层 merge,不在库层撞车
改完 GUI → 自动生成
// 自动产出,无需手写
Schema::table('users', function (Blueprint $t) {
    $t->renameColumn('user_name', 'username'); // 保数据
    $t->string('email', 128)->unique();
});

参数零手填,表单所见即所得

Collaboration

报 bug,不该靠在群里追问

bug 协作要的是沉淀和闭环,不是即时。这是 todos × Chrome 扩展真正改变协作的地方。

在微信群里报 bug
  • ✕ 一张截图 + 口头"点了没反应"
  • ✕ 开发追问"什么请求 / 报什么错"
  • ✕ 消息被聊天淹没、翻不到
  • ✕ 没指派、没状态、不了了之
用 todos × Chrome 扩展
  • ✓ 一键截图标注 / 录屏 + 自动打包现场
  • ✓ 带失败请求 / JS 报错 / DOM / 环境
  • ✓ 进待办:优先级、指派、完成闭环
  • ✓ 处理进度可回查,结构化、不丢

Engineering quality

不是 demo,是经得起改的工程

自用工具最怕"能跑就行、一改就崩"。所以质量这块是认真做的 —— 这些数字是"我对它有多大把握"的依据,不是营销。

红 → 绿回归

每个 bug 修复都配一个"先红后绿"的回归测试(先还原使其变红,再修复证明锁住),杜绝顺着现状写的假测试。

Philosophy

贯穿始终的几条取舍

单一事实源

一份 YAML 同时派生校验、文档、调试参数、表单控件 —— 改一处,四端同步,天然不漂移。

功能克制

业务面广,但每块功能都刻意简单 —— 能走 git 的,就不自己造版本/撤销/审计。

dev 写 · prod 只读

高风险写操作只在开发环境开放。正因为有这条边界,工具才不需要复杂的权限模型。

AI · MCP 桥

线上错误经 MCP 拉回代码仓库修复、再回写已解决 —— 让"看"和"建"在 AI 层闭环。

几句实话(边界不藏)

  • — 这是自用工具、强约定、深度耦合:赢在"和自己代码零漂移",代价是出了这个生态不一定适用。
  • — 部分是私有/商业包(scaffold 未上 Packagist、system 商业授权),装起来有前置约定。
  • — 前端监控对标 Sentry 核心,但 Web Vitals、录屏回放、会话存档没做。
  • — 部分仍早期:radar 全量群监控待二期,Chrome 扩展待上架;自托管需自运维。
